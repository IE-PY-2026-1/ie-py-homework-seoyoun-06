# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 나의 소개팅 애프터 성공률 분석 시스템
<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 사용자가 소개팅 후, 상대방과의 다양한 상호작용 데이터를 입력받아, 이를 분석하여 애프터 성공 확률을 예측해주는 데이터 기반 소개팅 애프터 분석 시스템입니다.
* 소개팅 종료 후 발생하는 비정형 데이터(연락 답장 속도, 이모티콘 사용 빈도, 실제 대화 시간 등)를 변수로 입력받아 애프터 성공 확률을 도출합니다.
* 1차 버전에서는 기초 변수 입출력을 통해 '애프터 지수'를 산출하며, 향후 2차(조건문 기반 등급 판정), 3차(메뉴 시스템 구축), 4차(데이터 고도화 및 다중 관리) 과정을 거쳐 체계적인 연애 의사결정 지원 도구로 확장할 계획입니다.
*최종적으로는 투입된 자원(시간, 비용) 대비 감정적 만족도의 효율성을 분석하여 최적의 애프터 시점을 제안하는 통합 오퍼레이션 시스템을 완성하고자 합니다.
<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 소개팅 상호작용 데이터 입력: 상대방 성함, 만남 지속 시간, 연락 답장 속도 등 기초 데이터 수집
2. 종합 애프터 성공 지수 산출: 입력된 변수들에 가중치를 적용하여 재회 가능성 수치화
3. 호감도 구간별 등급 판정: 분석된 점수를 바탕으로 등급부여
4. 최적 의사결정 및 보상 제안: 분석 결과에 따른 선연락 타이밍 추천 및 애프터 멘트 가이드
5. 연애 포트폴리오 전체 출력: 누적된 소개팅 데이터 리스트 및 분석 결과 조회
6. 시스템 종료: 데이터 세션

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
  * 이번 과제는 소개팅 후의 상대방과 비정형적인 데이트를 사용자로부터 입력받아서 리스트 안에 저장한 후에 가중치로 설계된 로직을 통하여 애프터 성공 확률을 산출하는 지능형 데이터 기반 연애 분석시스템이다.
  * for() 문을 활용하여 데이터를 순차적으로 입력받은 후에 각각의 데이터들을 리스트 안에 넣어 결론적으로 다시 전체 포트폴리오 리스트에 중첩하여 저장.
  * 가중치 공식은 (대화시간 * 2.0) + (이모티콘 빈도 * 0.3) + ((10 - 답장속도) * 0.3) 사용하여 10점 만점을 기준으로 정규화하여 최종결과를 산출하는 형태로 구성.
  * if-elif-else를 사용하여 S부터 D등급까지 총 5단계 등급으로 산출되도록 구현.
  * and 연산자를 사용해 특정상황에서 조건을 만족할 경우에 핑크빛 고속도로라는 문장이 산출되도록 설계하였고, break를 통하여 높은 점수를 얻을 경우에 분석을 미리 종료하는 프로그램을 설계함.
  * 리스트 조작 함수들을 사용해 데이터의 통계적 분석.
  * min(), max() 함수를 활용해 최종결과산출 값의 한계선 정의 

    
### **🤖 AI 파트너십 과정**
1. **내용 1: 일정한 방향성을 갖추기 위한 논리에 대한 질문**
    * **프롬프트 요약:** "답장 속도가 빠르면 빠를 수록 긍정적인 것이므로 점수를 총합을 낼 때 원하는 결괏값이 이상하게 나와. 어떻게 설계를 해야만 답장의 속도가 빠를 때, 즉 숫자가 작을수록 점수를 높게 나오게 할 수 있을지 알려줘"
    * **적용 내용:** AI가 제안한 역산 공식이라는 것을 통해 내가 원하는 결괏값이 나올 수 있도록 변경하여 설계함.이로써 모든 변수의 방향성을 하나로 통일시켜 논리도출.

2. **내용 2: 가중치부여방법에 대한 질문**
    * **프롬프트 요약:** "대화 시간이 이모티콘의 수보다 훨씬 중요하다고 생각하는바이므로 이런 것을 단순히 곱하기하여 중요도를 표현해낼 수 있을까?"
    * **적용 내용:** 각 변수에 각각 중요한 순위를 나눠 다른 상수를 곱하는 가중치 개념을 넣음. 가장 중요한 대화시간 부분에 2.0이라는 숫자를 포함시켜 더욱 세밀화시킴.
    
### **🛠️ Troubleshooting & 기술 회고:**
1. **문제 1: 2차원 리스트에서의 데이터 오류발생**
    * **원인:** portfolio 라는 리스트 안에 상대방의 정보리스트가 들어가있는 중첩된 구조여서 for 문법에서 데이터를 꺼내올 때 인덱스 번호가 헷갈리게 되어 이름, 점수가 섞여서 출력되는 현상이 발생함.
    * **해결:** person[0], person[1] 등 처럼 각각의 인덱스가 의미하는 것을 주석으로 작성하고 반복문 안에서 변수명에 다시 할당시켰음.
2. **문제 2: 데이터 비율의 오차**
    * **원인:** 사용자에게 입력받을 때, 답장속도 부분에서 1부터 10의 범위를 크게 벗어나는 숫자들을 입력할 시, 최종결과가 규칙선없이 비정상적이게 산출됨을 발견함.
    * **해결:** min(), max() 과 같은 함수를 사용하여 최대와 최소범위를 정하였음. 1부터 10안에서 고정되도록 설계하여 신뢰성을 더욱 올림.

## 🟨 [3차 과제: V3.0] 무한 루프와 메뉴 시스템 (반복문) - 향후 작성 예정
### **✨3차 과제 업데이트 내용:** 
  * 이번 3차과제에서는 기존의 2차과제에서 설계했던 지능형 데이터기반 연애 분석 시스템을 한 단계 발전시켜 구조적 함수와 무한 루프 메뉴 인터페이스를 탑재한 시스템을 구축하는 데에 초점을 두었다.
  * while Ture 무한 루프를 설계하여 사용자가 프로그램을 수동으로 종료하기 전, 즉 4번에 도달하기 전까지 지속하여 세션이 유지되는 4단 메인 메뉴 인터페이스를 구현하였다. 총 입력, 조회, 분석, 종료로 구성되었다.
  * 기존의 단일 흐름 코드를 역할에 따라 3개이상인 총 4개로 구별하여 서로 독립인 개별 함수로 분리하였고,메인 로직은 함수 호출위주의 목차식으로 수정하였다.
  * 핵심 연산 함수인 calculate_score()에 매개변수를 통하여 개별 상대방 리스트 데이터를 누락없이 저장시키고 데이터 입력함수 내부에서 global 키워드를 때와 맞게 사용하여 함수 종료 후 데이터가 전부 출력히 되도록 설계하였다.
  * 이전의 설계를 검토한 후에 분석 함수 내의 불필요한 조기 break 코드를 수정했고 메인 종료 메뉴 선택 시에만 break 문이 올바르게 작동되도록 하여 오류를  보완하였다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1** 논리의 추상화와 함수 분리방법에 관한 질문
    * **프롬프트 요약:**  기존의 소개팅 애프터 분석 코드를 3개이상으로 나누어야돼. 메인 while 루프 안을 목차처럼 깔끔하게 나누어 만든 후에, 가중치 계산 부분을 매개변수랑 return을 쓰는 독립적인 함수를 띄는 구조로 만들수 있는 방법을 알려줘.
    * **적용 내용:** AI가 제안한 논리 분리 방법을 따라 입력, 조회, 분석, 출력을 서로 독립된 영역으로 구분하여 기존의 설계내용을 수정

2. **내용2:** 데이터저장을 위한 전역변수와 global 키워드에 대한 질문
   * **프롬프트 요약:** 메뉴를 만들어서 1번을 선택하면 입력하고 2번을 선택하면 분석이 되는 시스템으로 만들고 싶어. 그렇기 위해서 함수가 끝날 때 입력한 데이터가 사라지지 않도록 전역변수를 안전하게 저장되도록 하는 방법과 global 키워드 위취 알려줘.
   * **적용내용:** 프로그램의 맨 꼭대기에 portfolio 라는 리스트를 전역 변수로 두고 데이터를 직접 수정하고 추가할 수 있는 input_date() 함수 내부에서만 global portfolio를 명시하여 함수 실행이 끝난다하더라도 포트폴리오 데이터가 완전히 저장되고 계속해서 누적되도록 설
      
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 입력받는 함수를 실행한 후에 데이터 초기화와 증발 현상 발생
     * **원인:** 초기 portfolio 리스트를 input_date() 함수 내부에서 지역변수로 다루거나 global 선언을 깜빡하여 메뉴 1번을 통하여 데이터를 입력해도 함수가 종료되었을 때, 지역 스코프가 사라져 데이터가 없어지는 현상을 발견
     * **해결:** 리스트선언을 함수의 밖으로 완전히 분리시키고 데이터를 조작하는 입력 함수 내부 상단에 global portfolio 라는 키워드를 명시하여 프로그램이 종료되기 전까지 누락없이 저장되도록 해결
    
2. **문제 2:** 특정 데이터를 분석할 때 리스트 출력시 전부 출력되지 않고 중간에 끊기는 현상 발생
   * **원인:** analyze_portfolio() 내부에서 특정점수인 9.5점을 초과할 시에 미리 종료되도록 설계해둔 break 문법이 실행되면서 첫 번째 데이터가 고득점이라면, 뒤의 나머지 소개팅 상대들의 데이터 분석이 실행되지 않고 반복문이 전부 누락되는 상황이 발생
   * **해결:** 필수 요구사항인 메인 메뉴 종료 탈출 기능에서만 break 가 작동되도록 하고, 분석 함수 내의 불필요한 조기 종료 break를 제거한 후에 안내문구로 대신하여 입력되어진 모든 데이터가 전부 누락없이 전수분석이 되도록 한 후에 통계적으로 산출되도록 설계를 수정

### 🟥 [4차 과제: V4.0] 모듈화 및 데이터 확장 (배열과 함수) - 🌟최종 완성 -- 향후 작성 예정
### **✨4차 과제 업데이트 내용:**
* 이번 4차 과제에서는 기존의 분석시스템에서 더하여 다수의 소개팅 데이터들을 체계적으로 분석하고 관리하여 결괏값을 산출해내는 통합형 데이터 시스템을 구축하여 마무리 하였다.
* 여러 상대방들의 답장속도, 대화 시간 등과 같은 비정형 데이터를 portfolio라는 이름의 2차원 리스트에 append 문법을 사용하여 누적하고 중첩 for문을 이용하여 통계 표로 간결하게 나타나도록 구현
* 분석된 데이터들의 프로그램이 종료되고, 누락되지 않도록 with open() 함수를 활용한 후, csv 확장자를 이용해 파일 형태로 저장하는 시스템을 구현
* 사용자의 입력에 있어서 오류가 발생할 시에 프로그램의 강제 종료를 방지하기 위해 예외처리와 같은 문법을 추가
* 대화시간과 이모티콘 빈도수, 답장속도 지연의 기준점의 합산들을 10점 만점으로 규정하여 종합 애프터 지수라는 정확한 수치로 도출하도록 설계
  
    
### **🤖 AI 파트너십 과정**
 1. **내용 1** 가독성 차원에서 간결히 나타낼 방안에 대한 질문
    * **프롬프트 요약:**  여러명의 데이터를 넣은 2차원 리스트를 print()로 계속해서 사용하니 괄호가 많이 보여 가독성이 떨어질 것 같다는 생각이 들었는데, 중첩 for 문을 활용해 깔끔하게 나타낼 방법 없을까?
    * **적용 내용:** AI가 제안한 데이터인 헤더라는 리스트와 중첩 for문, 파라미터를 활용하는 포맷팅 방법을 사용해 가독성을 더욱 올렸음.
2.  **내용 2** 예외처리와 적절한 파일 저장위치에 대한 질문
    * **프롬프트 요약:**  데이터를 csv로 저장하는 기능을 넣어야하는데, 또 input에서 사용자 입력을 받았을 때 에러가 발생하지 않도록 try except 문을 2개이상써야해 어느 부분에서 사용하면 좋을까?
    * **적용 내용:** 입력받는 함수인 input()의 내부에 float()와 int()와 같은 문자열을 형변환해주는 과정 속에 ValueError 와 같은 예외 처리 문법을 추가했고, 종료 직전 파일 쓰기를 수행하는 함수를 구축하여 오류방지
      
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 잘못 입력 시, 무한 루프에 빠지게 되는 현상 
     * **원인:** try except 문법에서 문자를 잘못 입력해서 에외가 발생할 때, 반복문을 탈출하지 못 하고 계속해서 같은 질문을 반복해서 받는 경우나 프로그램이 멈추는 현상을 발견함.
     * **해결:** While True 문 안에서 정상적인 숫자가 입력되고 형변환으로 성공했을 때에만 break 가 활성화되어 탈출이 되도록 코드를 수정

 2. **문제 2:** 파일 저장시에 글자 깨짐 현상
     * **원인:** open() 함수로 파일저장시, 인코딩 설정이 맞지 않아 csv 파일 내의 이름 데이터가 깨지는 것을 발견
     * **해결:** encoding="utf-8" 을 추가하여 글이 깨지지 않도록 함
